Man spricht von 

== Basisklasse // Vaterklasse // Parent Class // Base class
== Kindklasse // child class // Abgeleitete Klasse // Derived Class

== Im Ganzen:

   - Eine Kindklasse ist eine Spezialisierung einer Vaterklasse.
   - Eine Vaterklasse stellt die Verallgemeinerung (Gerneralisierung) einer Kindklasse dar.

Durch die Vererbung werden alle Member-Variablen der Vaterklasse an die Kindklasse geerbt.


a) Wie gelangen Daten in eine Objekt: Durch Konstruktoren

b) Wie gelangen Daten in eine Basisklasse eines Objekts: Durch die Memberwise Initializer List ( : nach dem Konstruktor)

  i)  Man öffne die Basisklasse für Kindklassen: protected
  ii) (private): Ich benötige einen Basisklassenkonstruktor, den ich in der Memberwise Initializer List

Virtuelle Methoden:

      Laufzeit entscheidet  // Late-Binding // Spätes Binden der Methodenadresse // Indirekter Methodenaufruf
	                        // Runtime // (Langsamere Ausführung)

Nicht-Virtuelle Methoden:

      Übersetzungszeit entscheidet // Early-Binding // Frühes Binden der Methodenadresse // Compile-Time // (Schnelle Ausführung)

virtual:

== Kennzeichnet Methoden in Basisklassen
== Diese haben eine Realisierung
== Diese Realisierung kann in Kindklassen geändert / erweitert werden.


i) virtual ist eine Einladung, dass diese Methode in der Kindklasse überschrieben wird. // muss aber nicht sein.

ii) non-virtual: Diese Methode wird an alle Kinkobjekte ver-erbt. Es gibt keinen Grund für eine Änderung dieser Methode.
