A) Was gibt es in C?

   Adressen (Address) und Zeiger (Pointer).

   int*   : Beschreibt einen Datentyp: Es ist ein Adress Datentyp.

   Beispiel:  0x00000084f3eff 864

   Was liegt hinter einer Adresse // Wieviele Daten liegen hinter einer Adresse? 1 Byte

   Ein int belegt 4 Bytes:

   0x00000084f3eff 864 beschreibt das ERSTE Byte einer int-Variablen // Anfangsadresse
   0x00000084f3eff 865
   0x00000084f3eff 866
   0x00000084f3eff 867

   Adresse des nächsten, frei verfügbaren Speichers:
   0x00000084f3eff 868

   Nebenbemerkung: Was sind das für Bytes zwischen 2 Variablen:

	0x0000006E796FF318  cc cc cc cc  ÌÌÌÌ
	0x0000006E796FF31C  cc cc cc cc  ÌÌÌÌ
	0x0000006E796FF320  cc cc cc cc  ÌÌÌÌ
	0x0000006E796FF324  cc cc cc cc  ÌÌÌÌ
	0x0000006E796FF328  cc cc cc cc  ÌÌÌÌ
	0x0000006E796FF32C  cc cc cc cc  ÌÌÌÌ
	0x0000006E796FF330  cc cc cc cc  ÌÌÌÌ
	0x0000006E796FF334  cc cc cc cc  ÌÌÌÌ

	Das sind zum Zwecke des Testens / Fehlersuchens so genannte "Schattebytes" / Shadow Bytes:
	Whyyyyyyyyy: Wenn einer dieser cc-Werte überschrieben wird, dann bricht der Debugger die Ausführung des Programms ab !!!

Stack / Heap

Stack: Ist für Parameter zuständig

a) Wie sieht die Parameter-Übergabe im Detail aus ???

   Da gibt es mehrere Optionen:

   i) Call-by-Value       // Call-by-Copy

   ii) Call-by-Address    // Das ist ein C-Sprachmittel (damit auch in C++ verfügbar)

   ii) Call-by-Reference  // Das ist ein C++-Sprachmittel (nicht in C verfügbar)

         Hinter den Kulissen: Quasi dasselbe wie Call-by-Address
		 Schreibweise / Programmierung: Erheblich einfacher / besser lesbar

   Voreinstellung (Default) ist Call-by-Copy (Call-by-Value)


b) Wo liegen lokale Variablen im Speicher ???

   i) Auf dem Stack
   ii) Dieser wächst pro Unterprogramm-Aufruf in eine Richtung
   iii) Nach Rückkehr verkleinert es sich in der anderen Richtung
   iv) Wird durch Register wie den SP (Stack Pointer) verwaltet.

Topic: "Freundschaft"  // friendship

Beispiel:

bool operator== (const Time& lhs, const Time& rhs)
{
    if (lhs.m_hours == rhs.m_hours && lhs.m_minutes == rhs.m_minutes && lhs.m_seconds == rhs.m_seconds) {
        return true;
    }
    else {
        return false;
    }
}

A) Geht so nicht, weil operator == ist eine freie Funktion: Diese hat keine Möglichkeit,
auf private Member Variablen zuzugreifen.

B) Hmmm, man könnte die Meinung haben, dass - in diesem Fall - der Operator == , auf 2 Time Objekt angewendet ,
   irgendwie doch zur Klasse Time gehören sollte:

   Das geht mit dem Schlüsselwort 'friend'

ACHTUNG: Mit friend werden private Member zu public Member.

Das sollte man sich genau überlegen: Ich würde friend nur in Ausnahmefällen einsetzen.
